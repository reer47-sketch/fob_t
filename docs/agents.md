# FOBreeders 자동 업데이트 규칙

기능 추가 또는 수정 작업이 완료될 때마다 아래 3가지를 **항상** 수행합니다.
사용자가 별도로 요청하지 않아도 자동으로 수행하세요.

---

## 1. docs/support.md 업데이트

AI 어시스턴트(`/api/assistant/chat`)의 시스템 프롬프트 파일입니다.

업데이트 대상:
- `## FOBreeders 앱 기능 전체 설명` — 신규/변경된 기능 설명 반영
- `## AI가 직접 수행할 수 있는 작업` — AI가 처리 가능한 작업 목록 추가/수정
- `### action.payload 형식` — 신규 액션 타입 및 payload 형식 추가
- `## UI 메뉴 명칭 매핑` — 신규 페이지 경로가 있으면 추가

---

## 2. src/app/(client)/(desktop)/help/page.tsx 업데이트

앱 내 사용자 도움말 페이지입니다.

업데이트 대상:
- 신규 기능에 해당하는 섹션 내용 추가 또는 수정
- `AI 음성 도우미` 섹션의 기능 카드(`grid` 내 Card 목록) 최신화
- `SECTIONS` 배열에 신규 메뉴 항목이 생겼으면 추가

---

## 3. Notion 업데이트

FOBreeders 기능 업데이트 로그 페이지에 변경 내용을 추가합니다.

- Notion 페이지 ID: `34a44e03-a878-8185-a05f-e1f2dc972bc1`
- Notion 페이지 URL: https://www.notion.so/34a44e03a8788185a05fe1f2dc972bc1
- 추가 형식:
  ```
  ### YYYY-MM-DD — 변경 요약 제목
  
  #### 버그 수정 / 신규 기능 / 개선
  - 변경 내용 설명
  - 관련 파일: `경로/파일명`
  ```

---

## 적용 시점

- 코드 변경과 문서 업데이트는 **같은 커밋**에 포함하는 것을 원칙으로 합니다.
- 작은 버그 수정이라도 사용자에게 노출되는 기능이 바뀌면 반드시 수행합니다.
- 내부 리팩토링(로직 변경 없음)은 생략 가능합니다.
